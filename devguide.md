🧭 Compass Consulting - 프로젝트 기획 및 정의서

1. 프로젝트 핵심 철학 (Core Philosophy) 단순한 입시 기계 양산 NO: 학생의 본질을
   파악하고 스스로 방향을 찾게 만드는 '정의서(Definition)' 기반 컨설팅.

시간과 가치의 최적화 (하이브리드 토큰 경제):

🤖 AI 어드바이저 (1 토큰): 빠르고 즉각적인 피드백, 기본적인 방향성 제시 및
프롬프트 기반 첨삭.

🧑‍🏫 휴먼 컨설턴트 (100 토큰): 전문 인간 컨설턴트의 심층 첨삭, 최종 퀄리티 컨트롤.
(너의 시간 가치를 극대화)

2. 시스템 아키텍처 및 기술 스택 (Tech Stack) 1인 개발의 속도와 유지보수성을
   극대화하기 위한 모던 스택.

Frontend: Vite + React (TypeScript)

이유: 우리가 방금 세팅한 미친 속도의 개발 환경. SPA(Single Page Application)로
빠르고 매끄러운 사용자 경험 제공.

Backend & DB: Supabase (PostgreSQL, Auth, Storage, Edge Functions)

이유: 로그인, DB, 오디오/이미지 파일 저장, 서버리스 함수까지 한 번에 해결되는
올인원 백엔드.

AI & 3rd Party APIs:

Text/LLM: OpenAI (GPT-4o) 또는 Anthropic (Claude 3.5 Sonnet - 한국어 특화).

STT (음성 인식): OpenAI Whisper API (모의면접 답변 분석용).

TTS (음성 합성): ElevenLabs API 또는 OpenAI TTS (AI 면접관 목소리).

결제 모듈: 포트원(PortOne) 또는 토스페이먼츠 (토큰 충전용).

3. 극복해야 할 현실적인 허들 (Candor & Solutions) 🚨 생기부 OCR 환각 현상: 표
   구조와 워터마크 때문에 LLM이 텍스트를 망칠 확률이 높음.

해결책: 투트랙 전략. 클로바(Clova)나 구글 비전(Google Vision) API로 순수
텍스트를 먼저 뽑고, 그걸 LLM에 던져서 정제하기. (초기에는 유저가 텍스트를 직접
복붙하게 해서 개발 허들을 낮추는 것도 추천!)

🚨 휴먼 컨설팅 스케일링의 한계: 텔레그램으로 모든 걸 처리하면 나중에 감당 안 됨.

해결책: 텔레그램은 '알림(Notification)' 용도로만! 실제 첨삭과 답변 작성은 무조건
우리가 만들 웹 Admin 대시보드에서 처리하도록 구조화.

🚨 AI 퀄리티 컨트롤: 뻔한 챗GPT 답변이 나오면 서비스 매력도 하락.

해결책: RAG(검색 증강 생성) 도입. 너의 과거 합격 사례, 세특 예시를 Supabase 벡터
DB(pgvector)에 박아두고, AI가 그걸 참고해서 답변하도록 세팅.
